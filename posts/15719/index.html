瀏覽器中頁面的生命週期

Author: DahisC Date: August 9, 2021  13:12:21

頁面的生命週期

在瀏覽器中，從使用者打開頁面到離開頁面的過程會經歷下述四個事件，分別為：

  1. DOMContentLoaded - DOM 掛載至頁面中
  2. load - 除了 DOM 之外，<img>, <style> 等已被載入（不包含<script>）
  3. beforeunload - 使用者按下關閉頁面
  4. unload - 使用者已關閉頁面

DOMContentLoaded

DOMContentLoaded 事件會在所有 DOM 已被掛載至頁面後觸發，但在這階段時還不包含外部引入的檔案，像是透過 <style> 引入的樣式檔以及 <img> 的圖片檔，也就是說在這階段很可能只會看到網頁原始不包含任何樣式的樣貌，但已經可以透過 JavaScript 操作 DOM，因為此時 DOM 已經被掛載至頁面中。

比較特別的是，在這階段如果 <body> 標籤的「中間」包含了一個 <script> 標籤，則瀏覽器會停下來先執行 <script> 標籤內部的程式碼或者請求檔案（如果 <script> 擁有 src 屬性）後再繼續執行 DOM 的生成。

上述情況有兩個例外：

  1. <script async>
  2. 以 JavaScript 的 createElement 方法插入的 <script>

load

load 會在頁面被完整繪製時觸發，其中也包含樣式檔以及圖片檔等等，在 load 之後的畫面就相當於要呈現給使用者的最終畫面，也是常用的 window.onload 的觸發階段。

beforeunload

beforeunload 事件會在使用者離開頁面時觸發（也就是已經按下了關閉分頁的 X 但尚未關閉分頁時）；一般會在這個階段詢問使用者是否要關閉分頁等等，讓使用者有最後反悔的機會。

unload

unload 事件會在使用者離開頁面後觸發，一般會在這個階段送出頁面的統計數據，但由於一般的 XMLHttpRequest 預設是異步模式，所以如果在此事件中使用異步的請求通常會被瀏覽器忽略，因為在執行時使用者已經關閉了頁面（也就相當於停止執行頁面上的 JavaScript），所以需要將 XMLHttpRequest 的 open() 方法第三個引數填入 true 開啟異步模式，就可以確保使用者在觸發此事件時先等待請求的結果後才關閉頁面；然而這個方式可能會導致使用者離開頁面的動作被暫時停止，因為同步執行的請求會阻塞使用者離開頁面的事件。
取而代之的是，如果透過 navigator.sendBeacon() 則是以異步的方式送出請求，卻又確保了能夠執行請求並不會阻塞使用者的離開頁面事件。
